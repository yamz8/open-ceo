# Board Deck Structure Guide

Detailed guidance for creating an effective board deck for VC-backed startups.

## Deck Overview

A typical board deck runs 20-40 slides, designed to be presented in 45-60 minutes with time for discussion. Structure content for both live presentation and async reading (board members often review beforehand).

## Section-by-Section Guide

### 1. Executive Summary (1-2 slides)

**Purpose:** Give board members the complete picture in 60 seconds.

**Content:**
- 3-5 key highlights from the quarter
- 1-2 key challenges or concerns
- Specific asks or decisions needed from the board
- Critical metrics snapshot

**Template:**
```markdown
## Executive Summary

### Highlights
- Closed $X in new ARR, exceeding target by Y%
- Launched [product feature], adopted by Z% of customers
- Hired [key role], bringing team to N people

### Challenges
- Sales cycle lengthening in enterprise segment
- Churn elevated in SMB cohort

### Board Asks
1. Approve [specific decision]
2. Introductions to [specific targets]
3. Input on [strategic question]
```

**Tips:**
- Lead with the most important information
- Be specific with numbers
- Don't bury bad news
- Asks should be actionable

### 2. Metrics Dashboard (2-3 slides)

**Purpose:** Visual snapshot of business health.

**Content:**
- Revenue metrics (ARR, MRR, growth)
- Unit economics (CAC, LTV, payback)
- Retention (NRR, churn)
- Cash position (burn, runway)

**Layout:**
- Use a consistent dashboard format each meeting
- Show current value, change from last period, trend
- Color code: green (on track), yellow (watch), red (concern)
- Include sparklines or mini-charts for trends

**Tips:**
- Keep format consistent meeting to meeting
- Board should be able to compare to previous decks easily
- Highlight metrics that changed significantly
- Include targets/plan for comparison

### 3. Business Update (3-5 slides)

**Purpose:** Progress narrative against strategic priorities.

**Content:**
- Progress on quarterly/annual OKRs
- Key initiatives status
- Major wins and milestones
- Timeline of significant events

**Structure for each priority:**
```markdown
## [Priority Name]

**Goal:** [What we set out to achieve]

**Status:** On Track / At Risk / Off Track

**Progress:**
- [Specific accomplishment with data]
- [Specific accomplishment with data]

**Next Steps:**
- [What's planned for next period]
```

**Tips:**
- Connect progress to strategic priorities set in previous meetings
- Use data to demonstrate progress, not just activities
- Be honest about what's off track and why

### 4. Financial Review (3-4 slides)

**Purpose:** Detailed financial performance and outlook.

**Content:**
- P&L summary (actual vs. plan)
- Revenue breakdown and trends
- Expense analysis
- Cash flow and runway
- Updated forecast if changed

**P&L Format:**
```
                    Actual    Plan    Variance
Revenue             $XXX      $XXX    +X%
COGS                $XXX      $XXX    +X%
Gross Margin        XX%       XX%     +X pts

Operating Expenses
  Sales & Marketing $XXX      $XXX    +X%
  R&D               $XXX      $XXX    +X%
  G&A               $XXX      $XXX    +X%
Total OpEx          $XXX      $XXX    +X%

Net Burn            $XXX      $XXX    +X%
```

**Tips:**
- Always show actual vs. plan
- Explain significant variances (>10%)
- Show runway scenarios if relevant
- Include forecast changes with rationale

### 5. Product & Engineering (2-3 slides)

**Purpose:** Product progress, roadmap, and technical health.

**Content:**
- Shipped features and releases
- Product metrics (adoption, engagement)
- Roadmap for next quarter
- Technical debt and infrastructure
- Engineering team velocity

**Roadmap Format:**
```
Now (This Quarter)    Next (Next Quarter)    Later (6+ Months)
├── Feature A         ├── Feature D          ├── Feature G
├── Feature B         ├── Feature E          ├── Feature H
└── Feature C         └── Feature F          └── Feature I
```

**Tips:**
- Focus on customer impact, not just features shipped
- Show adoption metrics for launched features
- Be realistic about roadmap timelines
- Surface any technical debt concerns

### 6. Go-to-Market (3-4 slides)

**Purpose:** Sales, marketing, and customer success performance.

**Content:**
- Pipeline and bookings
- Sales metrics (win rate, cycle time, ASP)
- Marketing performance (leads, conversion)
- Customer success (NPS, health scores)
- Notable wins and losses

**Sales Pipeline Format:**
```
Stage           Count    Value     Conversion
Qualified         XX     $XXX         -
Discovery         XX     $XXX        XX%
Evaluation        XX     $XXX        XX%
Negotiation       XX     $XXX        XX%
Closed Won        XX     $XXX        XX%
```

**Tips:**
- Include pipeline coverage ratio (3-4x of target is healthy)
- Highlight significant deals (won or lost) with learnings
- Show trends in key metrics (win rate, cycle time)
- Include competitive intelligence if relevant

### 7. Team & Organization (1-2 slides)

**Purpose:** Team growth, hiring, and organizational health.

**Content:**
- Current headcount by function
- Hiring progress vs. plan
- Key hires and departures
- Organizational changes
- Culture and engagement metrics

**Org Table:**
```
Function        Current    Plan    Open Roles
Engineering        XX       XX         X
Sales              XX       XX         X
Marketing          XX       XX         X
Customer Success   XX       XX         X
G&A                XX       XX         X
Total              XX       XX         X
```

**Tips:**
- Highlight key person risks
- Note any regrettable departures
- Include hiring velocity if relevant
- Surface any culture concerns proactively

### 8. Strategic Discussion (2-4 slides)

**Purpose:** Deep-dive on 1-2 key topics requiring board input.

**Format:**
```markdown
## [Strategic Topic]

### Context
[Why this matters now]

### Options
1. **Option A:** [Description, pros, cons]
2. **Option B:** [Description, pros, cons]
3. **Option C:** [Description, pros, cons]

### Recommendation
[Your recommended approach and rationale]

### Discussion Questions
1. [Specific question for board]
2. [Specific question for board]
```

**Good Topics:**
- Market expansion decisions
- Pricing strategy changes
- M&A opportunities
- Fundraising timing
- Major product pivots
- Competitive response

**Tips:**
- Come with a recommendation, not just options
- Frame clear questions for the board
- Provide enough context for informed discussion
- Limit to 1-2 topics to allow depth

### 9. Asks & Discussion (1 slide)

**Purpose:** Clear, actionable requests from the board.

**Content:**
- Specific decisions needed
- Introductions requested
- Expertise needed
- Upcoming decisions to flag

**Format:**
```markdown
## Board Asks

### Decisions Needed Today
1. Approve [budget/hire/initiative]
2. Ratify [option grant/policy change]

### Introductions Requested
- [Board Member]: Intro to [Company/Person] for [Purpose]
- [Board Member]: Intro to [Company/Person] for [Purpose]

### Expertise Needed
- [Topic area] - seeking advice on [specific question]

### Heads Up for Next Meeting
- Will bring [decision] for approval
- Expect update on [initiative]
```

**Tips:**
- Be specific about what you need
- Assign asks to specific board members when possible
- Include timeline if relevant
- Don't be shy about asking for help

### 10. Appendix (Variable)

**Purpose:** Backup detail for questions and reference.

**Content:**
- Detailed financial statements
- Full metrics history
- Cohort analysis
- Customer list or logos
- Competitive landscape detail
- Org chart
- Additional data supporting main deck

**Tips:**
- Don't present appendix slides
- Reference them when questions arise
- Keep organized and numbered for easy reference
- Update regularly but don't obsess over formatting

## Deck Design Principles

### Visual Consistency
- Use consistent colors, fonts, and layouts
- Template each slide type
- Board should recognize your format instantly

### Data Visualization
- One key message per chart
- Use appropriate chart types:
  - Trends → Line charts
  - Composition → Pie/stacked bar
  - Comparison → Bar charts
  - Distribution → Histograms
- Label clearly, minimize legends
- Always show trend direction

### Slide Layout
- Maximum 6 bullet points per slide
- Use headlines that convey the takeaway
- Include source and date on data slides
- White space is your friend

### Writing Style
- Active voice, present tense
- Specific numbers, not vague claims
- Headlines that tell the story
- Consistent terminology

## Pre-Meeting Checklist

Before sending the deck:
- [ ] All numbers are accurate and sourced
- [ ] Deck tells a coherent story
- [ ] Key messages are clear
- [ ] Asks are specific and actionable
- [ ] Bad news is addressed proactively
- [ ] Appendix has supporting detail
- [ ] Deck is consistent with previous format
- [ ] Someone else has reviewed for errors
